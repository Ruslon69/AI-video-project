# Session Start

Before coding in this repository:

1. Read all relevant files in `.ai/`, starting with `README.md` and `MASTER_PROMPT.md`.
2. Read durable project context in `docs/VISION.md`, `docs/DEVELOPMENT_WORKFLOW.md`, `docs/DECISIONS.md`, and `docs/TECHNICAL_DEBT.md` when the sprint touches product direction, workflow, architecture, or known debt.
3. Inspect the current repository state, including `git status --short`, existing docs, and the files related to the sprint.
4. Follow the engineering rules in `.ai/MASTER_PROMPT.md` and the relevant topic documents.
5. Identify conflicts between the sprint request, current code, and `.ai` rules before making changes.
6. Wait for the sprint description after reading these documents unless the user has already provided it.

Do not create a git commit unless explicitly requested.

# Session Start

Before coding in this repository:

1. Read all relevant files in `.ai/`, starting with `README.md` and `MASTER_PROMPT.md`.
2. Inspect the current repository state, including `git status --short`, existing docs, and the files related to the sprint.
3. Follow the engineering rules in `.ai/MASTER_PROMPT.md` and the relevant topic documents.
4. Identify conflicts between the sprint request, current code, and `.ai` rules before making changes.
5. Wait for the sprint description after reading these documents unless the user has already provided it.

Do not create a git commit unless explicitly requested.
